今天我们来学习 **AI Agent 的另一个核心设计模式 —— Reflection（反思）**

> 一句话核心：**Reflection** 就是让 AI 学会"自我检讨"——做完任务回头看看自己做得怎么样，哪里可以改进，然后迭代优化。

说实话，这个模式特别有意思。它像极了咱们工作中写代码的场景：写完一段代码 run 一下，发现报错了，回头检查检查，改改再跑……Reflection 就是让 AI 学会这套"写-跑-改"的循环。

之前我们学的 ReAct 是"边想边做"，而 Reflection 是"做完再想"。一个是过程优化，一个是结果优化。两个结合起来，AI 就能既聪明又靠谱。

<!-- more -->

# 是什么

先通过一张图来理解 Reflection 的工作原理：

![Reflection 工作原理](/assets/img/ailearn/daily/33/1.png)

Reflection 的核心流程就三步：

- **生成（Generate）**：让 AI 完成一个任务，输出初始结果
- **反思（Reflect）**：让 AI 回头审视自己的输出，找出问题和改进点
- **优化（Refine）**：根据反思的结果，重新生成更优质的输出

这个过程可以循环多次，直到质量满意为止。

举个我自己用的例子，让 AI 写一段 Python 代码：

- **第一次生成**：AI 写了个能跑的版本，但逻辑有点乱
- **反思阶段**：AI 回头看，发现自己的代码缺少错误处理，变量命名也不规范
- **优化阶段**：重写一版，加了 try-except，变量名也改得更有意义了

你看，这就像是一个有经验的程序员在 code review 自己的代码。

# 怎么做

光说不练假把式，下面看看怎么实现 Reflection 模式。

## 核心逻辑

最简单的实现就是"两次调用模型"：

```python
def reflection_agent(query, max_iterations=3):
    # 第一次：生成初始答案
    initial_response = llm.chat(f"请回答：{query}")

    current_response = initial_response

    for i in range(max_iterations):
        # 第二次：反思
        reflection_prompt = f"""
        你刚才的回答是：
        {current_response}

        请从以下几个方面反思这个回答：
        1. 是否完整回答了问题？
        2. 有没有遗漏重要信息？
        3. 逻辑是否清晰？
        4. 有没有可以改进的地方？

        如果有改进空间，请指出具体问题。
        如果已经很好了，请回复"无需改进"。
        """

        reflection = llm.chat(reflection_prompt)

        # 如果反思认为已经很好，就结束
        if "无需改进" in reflection:
            break

        # 第三次：根据反思结果优化
        refine_prompt = f"""
        原始回答：{current_response}

        反思意见：{reflection}

        请根据反思意见，重新生成一个更好的回答。
        """

        current_response = llm.chat(refine_prompt)

    return current_response
```

核心思路就是**生成 → 反思 → 优化**的循环。

## 进阶版：结构化反思

上面的版本比较简单，实际用的时候可以让反思更结构化：

```python
class ReflectionAgent:
    def __init__(self, llm):
        self.llm = llm

    def generate(self, task):
        """第一步：生成初始结果"""
        prompt = f"请完成以下任务：\n{task}"
        return self.llm.chat(prompt)

    def reflect(self, task, output):
        """第二步：结构化反思"""
        reflection_prompt = f"""
任务：{task}

你的输出：{output}

请按以下维度进行反思，输出 JSON 格式：
{{
    "completeness": "是否完整（1-5分）",
    "accuracy": "是否准确（1-5分）",
    "clarity": "是否清晰（1-5分）",
    "issues": ["问题1", "问题2"],
    "suggestions": ["建议1", "建议2"],
    "needs_improvement": true/false
}}
"""
        response = self.llm.chat(reflection_prompt)
        return json.loads(response)

    def refine(self, task, output, reflection):
        """第三步：根据反思优化"""
        refine_prompt = f"""
任务：{task}

当前输出：{output}

反思意见：
- 完整性评分：{reflection['completeness']}
- 准确性评分：{reflection['accuracy']}
- 存在的问题：{', '.join(reflection['issues'])}
- 改进建议：{', '.join(reflection['suggestions'])}

请重新生成一个更好的回答。
"""
        return self.llm.chat(refine_prompt)

    def run(self, task, max_iterations=3, quality_threshold=4):
        """运行完整流程"""
        output = self.generate(task)

        for i in range(max_iterations):
            # 反思
            reflection = self.reflect(task, output)

            # 检查是否达到质量要求
            scores = [
                reflection['completeness'],
                reflection['accuracy'],
                reflection['clarity']
            ]
            avg_score = sum(scores) / len(scores)

            if not reflection['needs_improvement'] or avg_score >= quality_threshold:
                print(f"达到质量要求，停止迭代。平均分：{avg_score}")
                break

            # 优化
            print(f"第 {i+1} 轮优化，当前平均分：{avg_score}")
            output = self.refine(task, output, reflection)

        return output
```

这样好处是**有量化的评分标准**，可以设定质量阈值，达标就停止，避免过度迭代浪费 token。

## 实战案例：代码生成与优化

来看一个实际的例子，用 Reflection 模式来写代码：

````python
import json
from openai import OpenAI

client = OpenAI()

class CodeReflectionAgent:
    def __init__(self):
        self.client = client

    def generate_code(self, requirement):
        """生成初始代码"""
        prompt = f"""你是一个 Python 专家。请根据以下需求编写代码：

需求：{requirement}

要求：
1. 代码要完整可运行
2. 包含必要的注释
3. 处理边界情况

请直接输出代码，不要解释。"""

        response = self.client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        )
        return response.choices[0].message.content

    def reflect_code(self, requirement, code):
        """反思代码质量"""
        prompt = f"""请作为一位资深代码审查员，审查以下代码：

需求：{requirement}

代码：
```python
{code}
````

请从以下维度进行审查，返回 JSON 格式：
{{
    "correctness": {{"score": 1-5, "comment": "是否正确实现了需求"}},
"readability": {{"score": 1-5, "comment": "可读性如何"}},
"robustness": {{"score": 1-5, "comment": "是否处理了异常情况"}},
"efficiency": {{"score": 1-5, "comment": "效率如何"}},
"issues": ["问题 1", "问题 2"],
"improvements": ["建议 1", "建议 2"],
"needs_rewrite": true/false
}}"""

        response = self.client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        )
        content = response.choices[0].message.content
        # 提取 JSON 部分
        if "```json" in content:
            content = content.split("```json")[1].split("```")[0]
        return json.loads(content)

    def refine_code(self, requirement, code, reflection):
        """根据反思优化代码"""
        improvements = "\n".join([f"- {imp}" for imp in reflection['improvements']])

        prompt = f"""请根据审查意见重写代码：

需求：{requirement}

当前代码：

```python
{code}
```

审查意见：
{improvements}

请输出改进后的完整代码。"""

        response = self.client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        )
        return response.choices[0].message.content

    def run(self, requirement, max_iterations=3):
        """运行完整流程"""
        print("=" * 50)
        print("🚀 开始代码生成与优化")
        print("=" * 50)

        code = self.generate_code(requirement)
        print(f"\n📝 初始代码已生成")

        for i in range(max_iterations):
            print(f"\n🔄 第 {i+1} 轮代码审查...")

            reflection = self.reflect_code(requirement, code)

            # 计算平均分
            scores = [
                reflection['correctness']['score'],
                reflection['readability']['score'],
                reflection['robustness']['score'],
                reflection['efficiency']['score']
            ]
            avg_score = sum(scores) / len(scores)

            print(f"   正确性: {reflection['correctness']['score']}/5")
            print(f"   可读性: {reflection['readability']['score']}/5")
            print(f"   健壮性: {reflection['robustness']['score']}/5")
            print(f"   效率: {reflection['efficiency']['score']}/5")
            print(f"   平均分: {avg_score:.1f}/5")

            if not reflection['needs_rewrite'] or avg_score >= 4.5:
                print(f"\n✅ 代码质量达标，停止优化")
                break

            if reflection['issues']:
                print(f"\n⚠️ 发现的问题：")
                for issue in reflection['issues']:
                    print(f"   - {issue}")

            print(f"\n🔄 正在优化代码...")
            code = self.refine_code(requirement, code, reflection)

        print(f"\n✨ 最终代码生成完成！")
        return code

# 测试代码生成

agent = CodeReflectionAgent()

requirement = """
写一个函数，接收一个文件路径，统计文件中各单词出现的频率，
返回出现频率最高的前 10 个单词及其次数。
要求：

1. 处理文件不存在的情况
2. 忽略大小写
3. 过滤掉常见的停用词（如 the, a, an 等）
   """

final*code = agent.run(requirement)
print("\n" + "=" * 50)
print("📄 最终代码：")
print("=" \_ 50)
print(final_code)

```

运行这个代码，你会看到 AI 先生成一版代码，然后自己审查一遍，发现问题再改进，直到质量达标。

有几个实用技巧：

- **评分维度要具体**：不同任务关注不同维度（代码关注正确性/可读性，文案关注流畅度/吸引力）
- **设定停止条件**：可以按迭代次数，也可以按质量分数，避免死循环
- **反思和生成用不同 prompt**：让反思更批判性，让生成更创造性

# ❄️ 冷知识

1. **Reflection 和 ReAct 是绝配**：ReAct 负责执行过程中的思考，Reflection 负责执行后的复盘，两者结合效果翻倍。

2. **Self-Refine 论文**：Reflection 模式的理论基础来自 2023 年的论文《Self-Refine: Iterative Refinement with Self-Feedback》，证明了让 AI 自我改进能显著提升输出质量。

3. **不是所有任务都需要反思**：简单问题直接回答就行，复杂任务（代码、写作、分析）才值得花额外 token 做反思优化。

4. **反思也可以外包**：除了让 AI 自己反思，还可以让另一个 AI（或人工）来做审查，有时候"旁观者清"效果更好。

5. **LangChain 有现成实现**：LangChain 提供了 ` ConstitutionalChain` 等组件，可以方便地实现类似 Reflection 的功能。
```
