今天我们来学习 **AI Agent 的设计模式 —— Router（动态路由模式）**

> 一句话核心：**Router**（动态路由模式）就像是一个智能客服转接系统，根据用户输入自动判断应该由哪个专门的 Agent 来处理，把请求路由到最适合的处理者手中。

说实话，这个模式太像我们日常生活中的场景了。你有没有打过客服电话？按下不同的数字键——1 号是查询余额，2 号是办理业务，3 号是投诉建议，0 号是转人工。这就是最朴素的路由思想。

在 AI 的世界里，Router 模式也是类似的道理：面对五花八门的用户请求，我们不能指望一个全能的 Agent 搞定所有事情，而是让一个"调度员"（Router）先分析用户意图，然后把请求分配给专门处理这类问题的 Agent。

它的核心价值在于**专业化分工和效率提升**——让每个 Agent 只做自己擅长的事，避免一个通用 Agent 什么都懂但什么都不精的尴尬局面。

# 是什么

老规矩，先看图解：

![Router 动态路由模式工作原理](/assets/img/ailearn/daily/35/1.png)

从图里能看到，Router 模式有三个核心组件：

- **Query（查询）**：用户的输入请求，比如"怎么退款"、"我要买会员"、"系统报错怎么办"
- **Router/Selector（路由器）**：智能调度中心，负责理解用户意图并决定应该转给哪个 Agent
- **Specialized Agents（专用 Agent）**：各个领域的专家，如客服 Agent、技术支持 Agent、销售 Agent 等

整个流程是这样的：

1. **接收用户请求**：用户说"我想退掉昨天买的东西"
2. **意图识别**：Router 分析这句话，识别出属于"售后/退款"类别
3. **路由决策**：Router 把请求转发给专门处理退款的 Customer Service Agent
4. **专业处理**：Customer Service Agent 用自己的专业知识回答用户
5. **返回结果**：最终答案返回给用户

举个我折腾的真实例子：

之前我搭建一个企业智能助手，最初的版本只有一个通用 Agent，什么问题都丢给它。结果你发现，问技术问题它能答，问 HR 政策它也能答，但质量参差不齐。问技术细节时，回答得不够深入；问公司福利时，又容易把政策说错。

后来改用 Router 模式：

- **技术问题** → 路由给 Tech Support Agent（技术专家）
- **HR 相关问题** → 路由给 HR Agent（人事专员）
- **销售咨询** → 路由给 Sales Agent（销售顾问）
- **其他问题** → 路由给 General Agent（通用助手）

效果立竿见影，每个 Agent 只专注自己的领域，回答质量大幅提升！这就是 Router 模式的威力：**专业的事交给专业的人来做**。

## 核心架构

Router 模式的核心代码结构：

```python
class RouterAgent:
    def __init__(self):
        self.router = Router()  # 路由决策器
        self.agents = {
            "tech_support": TechSupportAgent(),
            "customer_service": CustomerServiceAgent(),
            "sales": SalesAgent(),
            "general": GeneralAgent()
        }

    def process(self, user_query):
        # Step 1: Router分析用户意图，决定路由到哪个Agent
        target_agent_name = self.router.classify(user_query)

        # Step 2: 获取对应的Agent
        agent = self.agents.get(target_agent_name)

        # Step 3: 将请求转发给专门的Agent处理
        response = agent.handle(user_query)

        return response
```

关键点在于：

- **Router 负责分类**：准确理解用户意图，判断属于哪一类问题
- **Agent 专精分工**：每个 Agent 只处理自己领域的问题
- **易于扩展**：新增一个业务领域，只需新增一个 Agent 和路由规则
- **故障隔离**：某个 Agent 出问题不会影响其他 Agent

from typing import Dict, List, Optional, Callable
from dataclasses import dataclass
from enum import Enum
import json

# 模拟 LLM 客户端
class MockLLM:
    def generate(self, prompt: str, system_prompt: str = "") -> str:
        """模拟 LLM 生成响应"""
        # 实际场景中这里调用真实的LLM API
        return f"[LLM] 基于以下输入分析：{prompt[:80]}..."

llm = MockLLM()


class QueryCategory(Enum):
    """查询分类枚举"""
    TECH_SUPPORT = "tech_support"  # 技术支持
    CUSTOMER_SERVICE = "customer_service"  # 客服售后
    SALES = "sales"  # 销售咨询
    GENERAL = "general"  # 通用问题


@dataclass
class RoutingResult:
    """路由结果"""
    category: QueryCategory
    confidence: float  # 置信度 0-1
    reason: str  # 路由原因说明


class Router:
    """
    路由器：负责分析用户查询并决定转发给哪个Agent
    """

    def __init__(self):
        # 定义分类规则和关键词
        self.category_keywords = {
            QueryCategory.TECH_SUPPORT: [
                "bug", "error", "报错", "错误", "崩溃", "无法", "失败",
                "技术", "代码", "接口", "api", "数据库", "服务器",
                "部署", "配置", "环境", "日志", "调试"
            ],
            QueryCategory.CUSTOMER_SERVICE: [
                "退款", "退货", "换货", "售后", "客服", "投诉",
                "订单", "物流", "配送", "发票", "保修", "维修",
                "取消", "修改", "查询订单", "物流信息"
            ],
            QueryCategory.SALES: [
                "价格", "多少钱", "费用", "优惠", "折扣", "会员",
                "购买", "下单", "订购", "升级", "套餐", "方案",
                "咨询", "产品", "功能", "对比", "推荐"
            ]
        }

    def classify(self, query: str) -> RoutingResult:
        """
        分析用户查询，决定路由到哪个Agent

        实际项目中这里可以调用LLM进行意图识别
        """
        query_lower = query.lower()

        # 基于关键词匹配（简单实现）
        scores = {}
        for category, keywords in self.category_keywords.items():
            score = sum(1 for kw in keywords if kw in query_lower)
            scores[category] = score

        # 找出最高分的类别
        if scores:
            best_category = max(scores, key=scores.get)
            best_score = scores[best_category]

            # 计算置信度
            total_score = sum(scores.values())
            confidence = best_score / total_score if total_score > 0 else 0

            # 如果最高分为0或者置信度太低，归为通用问题
            if best_score == 0 or confidence < 0.3:
                return RoutingResult(
                    category=QueryCategory.GENERAL,
                    confidence=0.5,
                    reason="无法明确分类，使用通用Agent处理"
                )

            return RoutingResult(
                category=best_category,
                confidence=confidence,
                reason=f"关键词匹配成功，匹配到 {best_score} 个关键词"
            )

        return RoutingResult(
            category=QueryCategory.GENERAL,
            confidence=0.5,
            reason="默认分类"
        )

    def classify_with_llm(self, query: str) -> RoutingResult:
        """
        使用LLM进行更智能的分类（进阶版）
        """
        prompt = f"""
        请将用户查询分类到以下类别之一：
        - tech_support: 技术支持、bug报错、系统问题
        - customer_service: 退款、订单、售后服务
        - sales: 价格、购买、产品咨询
        - general: 其他通用问题

        用户查询: {query}

        请以JSON格式输出：
        {{
            "category": "类别名称",
            "confidence": 0.95,
            "reason": "分类理由"
        }}
        """

        # 模拟LLM返回
        response = llm.generate(prompt)

        # 这里实际应该解析LLM的JSON响应
        # 简化示例，根据关键词回退基础分类
        return self.classify(query)


class BaseAgent:
    """
    Agent基类：所有专用Agent的父类
    """

    def __init__(self, name: str, description: str):
        self.name = name
        self.description = description
        self.system_prompt = f"你是一个专业的{name}。{description}"

    def handle(self, query: str) -> str:
        """处理用户查询（子类必须实现）"""
        raise NotImplementedError

    def _call_llm(self, query: str) -> str:
        """调用LLM生成回答"""
        prompt = f"用户问题: {query}\n请给出专业、友好的回答。"
        return llm.generate(prompt, self.system_prompt)


class TechSupportAgent(BaseAgent):
    """技术支持Agent：处理bug、报错、技术问题"""

    def __init__(self):
        super().__init__(
            name="技术支持专家",
            description="专门处理软件bug、系统报错、技术配置等技术相关问题"
        )
        # 可以加载技术文档、FAQ等知识库

        }

    def handle(self, query: str) -> str:
        """处理技术问题"""
        print(f"  🔧 TechSupportAgent 处理中...")

        # 先检查是否是已知问题
        for error_code, description in self.knowledge_base["常见错误"].items():

        # 通用技术回答
        response = self._call_llm(query)
        return f"【技术支持】{response} 建议步骤：1）检查系统日志 2）确认配置正确 3）尝试重新安装"

class RouterSystem:
    """
    完整的路由系统：整合Router和各个Agent
    """

    def __init__(self):
        self.router = Router()
        self.agents: Dict[QueryCategory, BaseAgent] = {
            QueryCategory.TECH_SUPPORT: TechSupportAgent(),
            QueryCategory.CUSTOMER_SERVICE: CustomerServiceAgent(),
            QueryCategory.SALES: SalesAgent(),
            QueryCategory.GENERAL: GeneralAgent()
        }
        self.routing_history: List[Dict] = []

    def process(self, query: str, verbose: bool = True) -> str:
        """
        处理用户查询的完整流程

        Args:
            query: 用户输入
            verbose: 是否显示详细日志

        Returns:
            Agent的回答
        """
        if verbose:
            print(f"\n{'='*50}")
            print(f"📝 用户查询: {query}")
            print(f"{'='*50}")

        # Step 1: 路由分类
        if verbose:
            print(f"🔍 Step 1: Router 分析意图...")

        routing_result = self.router.classify(query)

        if verbose:
            print(f"   📊 分类结果: {routing_result.category.value}")
            print(f"   📈 置信度: {routing_result.confidence:.2f}")
            print(f"   💡 原因: {routing_result.reason}")

        # Step 2: 获取对应Agent
        agent = self.agents.get(routing_result.category)
        if not agent:
            agent = self.agents[QueryCategory.GENERAL]

        if verbose:
            print(f"\n🎯 Step 2: 路由到 {agent.name}")

        # Step 3: Agent处理
        response = agent.handle(query)

        # Step 4: 记录历史
        self.routing_history.append({
            "query": query,
            "category": routing_result.category.value,
            "confidence": routing_result.confidence,
            "agent": agent.name,
            "response": response
        })

        if verbose:
            print(f"\n✅ Step 3: 返回结果")
            print(f"{'='*50}")

        return response

    def batch_process(self, queries: List[str]) -> List[str]:
        """批量处理多个查询"""
        results = []
        for query in queries:
            result = self.process(query, verbose=False)
            results.append(result)
        return results

    def get_statistics(self) -> Dict:
        """获取路由统计信息"""
        if not self.routing_history:
            return {}

        category_counts = {}
        for record in self.routing_history:
            cat = record["category"]
            category_counts[cat] = category_counts.get(cat, 0) + 1

        total = len(self.routing_history)
        return {
            "总查询数": total,
            "分类分布": {k: f"{v/total*100:.1f}%" for k, v in category_counts.items()},
            "平均置信度": sum(r["confidence"] for r in self.routing_history) / total
        }

## 几个实用技巧

- **分层路由**：复杂的系统可以有多层路由，先分大类，再分小类
- **置信度阈值**：设置置信度阈值，低于阈值时转人工或要求 clarification
- **路由缓存**：对常见查询缓存路由结果，避免重复分类
- **A/B 测试路由**：测试不同路由策略的效果
- **人工接管**：不确定时提供人工客服选项
- **路由日志分析**：定期分析路由数据，优化分类准确度

# ❄️ 冷知识

1. **Router 模式的产品化案例**：OpenAI 的 GPTs Store 里，官方做了一个叫"ChatGPT"的 GPT，当你安装了多个 GPT 后，它会根据你的问题智能推荐应该使用哪个 GPT 来回答。这就是 Router 模式在产品层面的典型应用。

2. **API Gateway vs Router**：API Gateway（API 网关）和 Router 模式有几分相似。API Gateway 根据 URL 路径路由到不同微服务，Router 根据语义路由到不同 Agent。一个是基于规则，一个是基于理解，思想是相通的。

3. **LangChain 的 Multi-Route Chain**：LangChain 里有一个专门的`MultiRouteChain`，就是 Router 模式的官方实现。它通过 LLM 判断用户输入应该走哪条处理链路。

4. **Router 也可以递归**：在某些复杂场景下，Router 可以嵌套使用。第一层路由到"产品部门"，第二层再路由到"技术团队"或"销售团队"，形成一个树状的路由结构。

5. **Router 是构建 Agent 生态的基础**：想象未来的 AI Agent 商店，有成千上万个专业 Agent。Router 就是那个"应用商店"的入口，帮你找到合适的 Agent。没有 Router，Agent 再多也无法高效协同。
