# 文案编写指南

## 角色定位

**抖音教学运营专家**

- 擅长将复杂技术概念转化为通俗易懂的内容
- 深谙短视频平台的传播规律和用户习惯
- 注重内容的吸引力和完播率

## 文案结构

### 1. 标题设计

**原则**：

- 控制在 20 字以内
- 包含核心关键词
- 引发好奇心或紧迫感

**优秀示例**：

- "5 分钟搞懂 RNN：让 AI 拥有记忆的秘密"
- "LSTM vs GRU：谁才是循环网络之王？"
- "特斯拉放弃 Transformer 回归 GRU，原因竟然是..."

### 2. 开场白

**标准开场**：

```
"5分钟AI，每天搞懂一个知识点！今天我们学习，[主题名称]。"
```

**作用**：建立品牌认知，引导用户继续观看

### 3. 内容分段

**每段时长**：15-45 秒

**分段原则**：

- 一个场景 = 一个知识点
- 先结论后解释
- 避免信息过载

**结构模板**：

```
[观点/概念] → [解释说明] → [例子/比喻] → [总结/过渡]
```

### 4. 结尾收尾

**标准结尾**：

```
"感谢观看，若喜欢请关注，每天5分钟，轻松学AI"
```

## 写作技巧

### 通俗易懂

**错误示例**：
"RNN 通过隐藏状态的循环连接实现对序列数据的建模"

**正确示例**：
"RNN 就像分拣中心的传送带，持续传递包裹（信息），让每一步决策都能参考之前的内容"

### 数字精确化

**模糊表达**：
"LSTM 参数比 GRU 多不少"

**精确表达**：
"LSTM 参数比 GRU 多约 25%"

### 制造冲突感

**示例**：

- "2012 年 AlexNet 夺冠引爆深度学习，而 LSTM 论文同年发表却无人问津，直至 5 年后才成为 NLP 基石"
- "人脑处理一句话耗能约 0.3 卡路里，LSTM 耗能却是它的 1.2 万倍"

## 场景数量规划

| 视频时长 | 场景数  | 每场景时长 |
| -------- | ------- | ---------- |
| 2-3 分钟 | 5-6 个  | 20-30 秒   |
| 3-5 分钟 | 6-8 个  | 25-40 秒   |
| 5-8 分钟 | 7-10 个 | 30-45 秒   |

## 常见场景类型

1. **开场介绍**：引出主题，吸引注意
2. **概念定义**：核心概念解释
3. **原理讲解**：技术细节说明
4. **对比分析**：不同方案对比
5. **实际应用**：真实案例展示
6. **冷知识/趣闻**：增加趣味性
7. **结尾收尾**：总结回顾
