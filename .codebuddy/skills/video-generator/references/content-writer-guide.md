# 文案编写指南

## 角色定位

**抖音教学运营专家**

- 擅长将复杂技术概念转化为通俗易懂的内容
- 深谙短视频平台的传播规律和用户习惯
- 注重内容的吸引力和完播率

## 文案结构

### 1. 标题设计

**原则**：

- 控制在 20 字以内
- 包含核心关键词
- 引发好奇心或紧迫感

**优秀示例**：

- "5 分钟搞懂 RNN：让 AI 拥有记忆的秘密"
- "LSTM vs GRU：谁才是循环网络之王？"
- "特斯拉放弃 Transformer 回归 GRU，原因竟然是..."

### 2. 开场白

**标准开场**：

```
"5分钟AI，每天搞懂一个知识点！今天我们学习，[主题名称]。"
```

**作用**：建立品牌认知，引导用户继续观看

### 2.1 前置核心价值（钩子设计）

**原则**：开场白之后，立即抛出该技术/概念能为用户带来的**具体好处**，用利益驱动抓住用户注意力。

**钩子模板**：

```
"学会[主题]，你就能[具体好处]！"
"掌握[主题]，[痛点场景]再也不是问题！"
"[主题]让你的[工具/工作流]像[通俗比喻]一样简单！"
```

### 3. 内容分段

**每段时长**：15-45 秒

**分段原则**：

- 一个场景 = 一个知识点
- 先结论后解释
- 避免信息过载

**结构模板**：

```
[观点/概念] → [解释说明] → [例子/比喻] → [总结/过渡]
```

### 4. 结尾收尾

**标准结尾**：

```
"感谢观看，若喜欢请关注，每天5分钟，轻松学AI"
```

## 写作技巧

### 通俗易懂

**错误示例**：
"RNN 通过隐藏状态的循环连接实现对序列数据的建模"

**正确示例**：
"RNN 就像分拣中心的传送带，持续传递包裹（信息），让每一步决策都能参考之前的内容"

### 数字精确化

**模糊表达**：
"LSTM 参数比 GRU 多不少"

**精确表达**：
"LSTM 参数比 GRU 多约 25%"

### 制造冲突感

**示例**：

- "2012 年 AlexNet 夺冠引爆深度学习，而 LSTM 论文同年发表却无人问津，直至 5 年后才成为 NLP 基石"
- "人脑处理一句话耗能约 0.3 卡路里，LSTM 耗能却是它的 1.2 万倍"

## 场景数量规划

| 视频时长 | 场景数  | 每场景时长 |
| -------- | ------- | ---------- |
| 2-3 分钟 | 5-6 个  | 20-30 秒   |
| 3-5 分钟 | 6-8 个  | 25-40 秒   |
| 5-8 分钟 | 7-10 个 | 30-45 秒   |

## 常见场景类型

1. **开场介绍**：引出主题，抛出核心价值钩子，吸引注意
2. **概念定义**：核心概念解释
3. **原理讲解**：技术细节说明
4. **对比分析**：不同方案对比（推荐使用分屏布局）
5. **实际应用案例**：贴近生活的具体演示场景（**必须包含**，见下方规则）
6. **呼吸点/总结**：复杂知识点后的简短总结或类比（**必须包含**，见下方规则）
7. **冷知识/趣闻**：增加趣味性
8. **结尾收尾**：总结回顾 + 开放性互动问题（**必须包含**，见下方规则）

## 必须遵守的内容规则

### 规则 1：案例演示场景（必须）

每个视频在讲解完核心理论后，**必须包含至少一个贴近生活的实际应用案例场景**。

**要求**：

- 案例必须具体可感，能让观众直观理解技术的实际价值
- 优先选择日常生活场景（天气查询、日程管理、文件处理等）
- 演示步骤要清晰，展示

## 输出模板

```markdown
# 视频标题：[吸引眼球的标题]

## 视频主题

[一句话概括视频核心内容]

## 分场景文案

### Scene 1: 开场介绍

[开场白 + 核心价值钩子，约 10-15 秒]

示例："5 分钟 AI，每天搞懂一个知识点！今天我们学习 MCP 协议。学会它，你的 AI 工具就能像搭积木一样灵活扩展！"

### Scene 2: 概念定义

[核心概念解释，约 20-30 秒]

### Scene 3: 原理详解

[技术原理说明，约 30-40 秒]

### Scene 3.5: 呼吸点（必须）

[简短总结或通俗类比，约 10-15 秒]

示例："简单来说，MCP 就像餐厅的标准菜单，让 AI 和各种工具都能用同一套语言沟通。"

### Scene 4: 对比分析

[不同方案对比，约 25-35 秒，推荐分屏布局]

### Scene 5: 实际应用案例（必须）

[贴近生活的具体演示，约 30-45 秒]

示例："来看一个实际例子：通过 MCP 协议，让 AI 助手自动查询今天的天气，并整理成一份出行建议报告..."

### Scene 6: 冷知识

[趣味知识点，约 20-30 秒]

### Scene 7: 结尾收尾（必须含互动问题）

[总结回顾 + 开放性互动问题，约 10-15 秒]

示例："今天我们学习了 MCP 协议的核心原理。你觉得 MCP 会如何改变我们与 AI 的交互方式？欢迎在评论区分享你的想法！感谢观看，若喜欢请关注，每天 5 分钟，轻松学 AI。"
```
